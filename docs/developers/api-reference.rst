===============
# API Reference
===============

OpenBook currently maintains API reference material in two complementary forms:

1. Runtime OpenAPI documents generated by drf-spectacular.
2. Curated authentication flow specifications in ``doc/specs``.

This page explains how maintainers should use both sources together.

.. contents:: Page Content
   :local:


Runtime OpenAPI documents
-------------------------

When the local development server is running, OpenAPI outputs are available at:

- ``/auth-api/openapi.json``
- ``/auth-api/openapi.yaml``
- ``/auth-api/openapi.html`` (ReDoc)

Treat these endpoints as the canonical machine-readable contract for request
and response structures.


Curated flow specifications
---------------------------

The files in ``doc/specs/django-allauth-headless-api`` provide scenario-level
guidance and examples for authentication and account flows from a client
perspective.

Use them when you need practical behavior details such as:

- expected flow transitions after login/signup actions,
- CSRF handling patterns for browser clients,
- SAML/social login path details.


Documentation maintenance policy
--------------------------------

When API behavior changes:

1. Verify OpenAPI output reflects the implementation.
2. Update matching flow documentation in ``doc/specs``.
3. Add a user-visible changelog entry when behavior changes externally.
